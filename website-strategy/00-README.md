## Quick Context for the Developer

This site serves **three distinct audiences** — every design and content decision should keep this in mind:

| Audience | Primary Goal on Site | Key Pages |
|----------|---------------------|-----------|
| Families & Individuals | Learn about services, understand eligibility, request intake | Home, Services, How It Works, Contact |
| Service Coordinators / Regional Centers | Evaluate the agency as a vendor, make a referral | Referral Sources, Credentials, Make a Referral |
| Job Seekers (DSPs / Caregivers) | Learn about the role, apply | Careers, Benefits, Apply Now |

---

## Critical Design Notes

1. **Phone number** must be persistent in the header, click-to-call on mobile, on every page.
2. **Three CTAs** should be accessible from the homepage without scrolling: Get Started (families), Make a Referral (SCs), Join Our Team (caregivers).
3. **DDS Vendor Number** should appear on the Credentials page, the footer, and the Referral Sources page.
4. **No PHI ever** — the website should never store or transmit Protected Health Information. Intake forms collect name/contact only.
5. **Employee portal URL** (`/employee-portal/`) should be created and protected at launch even if the content is minimal — so the URL structure is established.
6. **Admin + Employee dashboards** are developer scope — see `06-employee-portal-planning.md` for full spec.
7. **Starting point is trinitychange.org** — replicate structure, revise content, improve everything per these docs.
